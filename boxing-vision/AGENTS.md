# Boxing Vision Game

## 컨셉
웹캠으로 플레이어의 실제 펀치 동작(잽/스트레이트/바디)과 회피(위빙)를 인식해서
1인칭 시점으로 가상 상대와 대전하는 복싱 게임.

## 스택
- Python
- OpenCV (웹캠 입력/프레임 처리)
- MediaPipe Pose (관절 좌표 추적 — 손목/팔꿈치/어깨/코 등)
- Pygame (게임 로직, UI, 1인칭 뷰 렌더링)

## 시점
1인칭. 플레이어 시야 = 상대 선수가 정면에 보이는 구도.
웹캠 화면은 코너에 작게 PIP(picture-in-picture)로 띄워서 본인 자세 확인 가능 (디버깅용, 나중에 옵션으로 끌 수 있게)

## 동작 인식 스펙 (MediaPipe 좌표 기반)

### 1. 잽 (Jab)
- 판정: 한쪽 손목이 어깨 높이 유지한 채 카메라 방향(z축, 또는 프레임 내 크기 변화)으로 빠르게 전진 후 복귀
- 특징: 빠른 속도, 짧은 궤적, 팔이 완전히 펴지지 않아도 인식

### 2. 스트레이트 (Straight/Cross)
- 판정: 반대쪽 손(주로 뒤손) 손목이 어깨~턱 높이에서 완전히 펴지며 전진
- 특징: 잽보다 느리지만 궤적 큼, 어깨 회전(양쪽 어깨 좌표 각도 변화) 동반 시 가중치

### 3. 위빙 (Weaving)
- 판정: 코(nose) 또는 머리 중심 좌표가 좌우로 일정 거리 이상 이동 + 상체 낮아짐(어깨-엉덩이 y좌표 간격 감소)
- 특징: 회피 판정 — 위빙 중에는 상대 공격 무효화 윈도우 발생

### 4. 바디 (Body shot)
- 판정: 손목이 어깨 높이보다 낮은 위치(가슴~복부 높이)에서 전진 동작
- 특징: 잽/스트레이트와 높이로 구분

## 판정 로직 우선순위
1. 프레임별 관절 좌표 버퍼링 (최근 N프레임, 예: 10~15프레임)
2. 속도/궤적/높이 조합으로 동작 분류 (규칙 기반 먼저, 필요하면 추후 간단한 임계값 튜닝)
3. 오탐 방지: 쿨다운(예: 0.3초) 걸어서 한 동작이 여러 번 중복 인식 안 되게

## 상대 AI
- 패턴 기반 (완전 랜덤 X, 정해진 패턴 로테이션 + 약간의 랜덤성)
- 상대도 잽/스트레이트/바디 공격 패턴 있고, 텔레그래프(공격 전 예고 모션) 존재
- 플레이어는 위빙으로 회피하거나 맞고 데미지 받음
- 라운드제 (예: 1라운드 60~90초) 또는 체력제 (상대 HP 0 → 승리, 플레이어 HP 0 → 패배)

## 스코프
- 1라운드, 상대 1명, 패턴 3~4개, 승패 엔딩
- 판정은 placeholder 도형(십자선/타겟 마커)으로 시작, 정교한 비주얼은 나중

## 퀄리티 기준
- 판정 반응속도 (렉 없이 실시간 인식되는 게 최우선 — 화려함보다 반응성)
- 화면 흔들림/히트스탑 (타격감)
- 판정 성공/실패 시각 피드백 (필수 — 안 그러면 왜 안 맞았는지 플레이어가 모름)

## 개발 순서
1. 웹캠 입력 + MediaPipe 관절 좌표 화면에 그리기 (동작 인식 없이 좌표만 확인)
2. 잽 하나만 판정 로직 구현 + 콘솔에 "JAB!" 출력해서 인식률 확인
3. 나머지 3개 동작(스트레이트/위빙/바디) 판정 추가
4. 상대 AI + 1인칭 UI 붙이기
5. 판정↔전투 로직 연결 (맞으면 데미지, 위빙하면 회피)
6. 승패 엔딩
